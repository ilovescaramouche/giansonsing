Background Property

★ A shorthand notation for all background properties.
★ Values are separated by spaces, not commas.

Background Color

  • Set color to a specific value (e.g., red)
  • Hexadecimal (e.g., #000000)
  • RGB values (e.g., rgb(255,0,0))
  • Percentage values (e.g., rgb(100%,0%,0%))
  • Inherit from parent (e.g., inherit)

Background Image

  • Set to an image (e.g., url("clouds.jpg"))
  • No background (none)
  • Inherit from parent (inherit)

Background Repeat

  • repeat – default
  • repeat-x – horizontal repeat
  • repeat-y – vertical repeat
  • no-repeat – no repetition
  • inherit – follow parent

Background Position

★ Position can be specified by values or percentages (e.g., 100px 50px, 25% 10%).
★ Keywords can also be used (e.g., top, center, bottom, left, right).

Background Attachment

  • scroll – background scrolls with document (default)
  • inherit – follow parent’s attachment
  • fixed – background stays fixed when scrolling

JavaScript Lessons Accordion

✪ L1: INTRO TO JAVASCRIPT ✪

✦ Script - a set of computer instructions or commands that puts together or connects existing components to accomplish a new related task

✦ JavaScript
○ It is a powerful client-side scripting language that is interpreted by the web browser.
○ It was created by Brendan Eich.
○ It enables web developers and designers to build more functional and interactive websites.
○ It was originally called Mocha, then Live Script, and finally JavaScript.
○ It is NOT associated with Java.
○ It is case-sensitive.
○ It is normally embedded in your HTML code.

Characteristics of JavaScript

  1. Interpreted Language – requires a browser to run
  2. Scripted Language – performs simple repetitive tasks
  3. Client-Side Scripting – executed on the user’s machine
  4. Object-Oriented Programming – treats every element as an object

JavaScript Terms

  • Objects – items that exist in the browser
  • Methods – actions performed on/by an object
  • Properties – subsections of an object
  • Event – something that happens (loading, clicking, hovering)
  • Parameter – data describing characteristics of an object
  • Type – variable type (integer, string, float, Boolean)
  • Type conversion – converting a variable from one type to another

L2: MORE ABOUT JS :: C.S

★ JavaScript is client-side, high-level, interpreted, and object-oriented.

  • Client-side: processed on the user’s computer
  • High-level scripting: written close to English
  • Interpreted: converted to machine code as it runs
  • Object-based programming language
  • Semi-colons optional but recommended

Some of the things JS can do:

  1. React to events
  2. Validate data
  3. Create cookies
  4. Enhance a webpage (pop-ups, animations, menus)

Features of JS:

  • Supports structured programming syntax in C
  • Dynamic typing
  • Runs locally for faster interaction
  • Detects user actions
  • Combines with CSS for DHTML effects

Disadvantages:

  • Some browsers don’t support JS
  • Secrets in JS can be extracted

L3: JAVASCRIPT SYNTAX

★ JS and DOM can be exploited by malicious authors.
★ Source code is sent to the client and can be reverse engineered.

JavaScript Terms

  • W3C – ensures compatibility in standards
  • DOM – interface to access/manipulate document contents
  • Parsing – analyzing syntax to extract information
